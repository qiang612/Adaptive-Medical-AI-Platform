## 二、存在问题及解决方案

### 2.1 主要问题

#### 问题一：批量推理接口未完全实现
**问题描述：**
在 `inference.py` 的 `/inference/batch` 接口中，当前使用 `random` 模块模拟诊断结果，未与 Celery 任务队列真正集成。这导致模型对比功能无法进行真实的异步推理。

**位置：** [backend/app/api/v1/inference.py#L30-50](file:///D:/Adaptive-Medical-AI-Platform/backend/app/api/v1/inference.py#L30-L50)

**解决方案：**
1. 修改接口逻辑，为每个模型创建 Celery 任务
2. 返回任务ID供前端轮询
3. 实现 WebSocket 实时推送任务完成通知

```python
# 建议修改代码示例
@router.post("/batch")
def batch_inference(req: dict = Body(...), ...):
    results = []
    for m_id in model_ids:
        # 创建异步任务
        task = run_inference_task.delay(model_id, patient_info, file_ids)
        results.append({
            "task_id": task.id,
            "model_id": m_id,
            "status": "pending"
        })
    return {"code": 200, "results": results}
```

#### 问题二：Worker节点状态监控为静态数据
**问题描述：**
任务监控页面中的 Worker 节点信息是硬编码的模拟数据，未接入真实的 Celery 集群状态 API。

**位置：** [backend/app/api/v1/tasks.py#L90-115](file:///D:/Adaptive-Medical-AI-Platform/backend/app/api/v1/tasks.py#L90-L115)

**解决方案：**
集成 Celery Inspector 获取真实集群状态：

```python
from celery_app import celery

@router.get("/workers")
def get_worker_nodes():
    inspect = celery.control.inspect()
    return {
        "active": inspect.active(),
        "stats": inspect.stats(),
        "registered": inspect.registered()
    }
```

#### 问题三：缺少WebSocket实时推送
**问题描述：**
当前任务状态更新依赖前端轮询，未实现 WebSocket 实时推送机制。

**解决方案：**
1. 添加 FastAPI WebSocket 支持
2. 前端建立 WebSocket 连接
3. Celery 任务完成后主动推送通知

#### 问题四：前端依赖库使用不一致
**问题描述：**
`package.json` 中引入 `js-cookie`，但实际代码使用 `localStorage`。

**位置：** [frontend/src/api/index.js](file:///D:/Adaptive-Medical-AI-Platform/frontend/src/api/index.js)

**解决方案：**
